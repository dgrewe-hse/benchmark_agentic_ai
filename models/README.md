# Models

This directory manages the model registry and quantization tooling for the benchmark lab.

## Contents

| File | Purpose |
|------|---------|
| `registry.yaml` | Pinned model list with HF repos, revision SHAs, vLLM config |
| `select_models.py` | Interactive script to populate `registry.yaml` |
| `quantize.sh` | Batch fp8/AWQ quantization via llm-compressor |

---

## Quickstart

### 1. Log in to HuggingFace

```bash
hf auth login
```

> **Note:** `huggingface-cli` is deprecated — always use `hf auth login`.

You need a HuggingFace account with a **read-only access token**.
Get one at: https://huggingface.co/settings/tokens

### 2. Accept gated model licenses

The following models require you to accept their license on the HuggingFace website
**while logged in** before you can download them:

| Model | License page |
|-------|-------------|
| Llama 3.2 3B | huggingface.co/meta-llama/Llama-3.2-3B-Instruct |
| Llama 3.1 8B | huggingface.co/meta-llama/Llama-3.1-8B-Instruct |
| Llama 3.3 70B | huggingface.co/meta-llama/Llama-3.3-70B-Instruct |
| Phi-4-mini | huggingface.co/microsoft/Phi-4-mini-instruct |

Click **"Agree and access repository"** on each page. The access token only needs
read permissions — write/admin access is never required.

### 3. Run the model selection script

```bash
source .venv/bin/activate
python3 models/select_models.py
```

The script will:
1. Verify your HF login
2. Display the full model catalog with tier, size, precision, and gated status
3. Let you select which models to include (by number, e.g. `1 3 5`, or `all`)
4. Fetch the current HEAD revision SHA from HuggingFace Hub for each model
5. Prompt before writing — confirm to overwrite `registry.yaml`

**Re-run any time** to add/remove models or refresh revision SHAs.

---

## Why revision SHAs?

Every entry in `registry.yaml` must have a pinned `revision` SHA. This is
non-negotiable for reproducibility:

- Without a SHA, `vLLM` loads whatever is currently at `HEAD` on HuggingFace
- Model weights can change silently between runs (bug fixes, re-uploads)
- Results become impossible to compare across time or machines
- Publication requires exact model versions

The SHA looks like: `a1b2c3d4e5f67890abcdef1234567890abcdef12`

To fetch one manually if the script fails:

```python
from huggingface_hub import model_info
print(model_info("Qwen/Qwen2.5-Coder-7B-Instruct").sha)
```

## Model Tiers

| Tier | Precision | Max Concurrent | VRAM Range |
|------|-----------|----------------|------------|
| Small (≤9B) | fp16 | 3 | 7–18 GB |
| Mid (8–14B) | fp16 | 2 | 19–32 GB |
| Large (16–32B) | fp8 | 2 | 24–34 GB |
| XL (70–72B) | fp8 | 1 (exclusive) | 76–78 GB |
| Reference | API | — | — (judge/evaluator only) |

> **Reference models** (GPT-4o, Claude) are used exclusively as **LLM judges**
> to evaluate challenge outputs. They are never benchmarked themselves.

---

## fp8 Models and Quantization

Large and XL tier models run at fp8 precision to fit in the DGX Spark's 128 GB
unified memory. Two options:

**Use a pre-quantized model from HuggingFace** *(recommended)*

Search for `fp8` variants on HuggingFace (e.g. `Qwen/Qwen2.5-Coder-32B-Instruct-FP8`)
and update the `hf_repo` field in `registry.yaml` accordingly.

## GLM-4 Models — Repo Verification Required

The GLM-4 family (`glm-4-air`, `glm-4-plus`) uses inconsistent naming on HuggingFace.
Before benchmarking, verify the correct repo ID:

```python
from huggingface_hub import HfApi
api = HfApi()
results = api.list_models(author="THUDM", search="glm-4")
for m in results:
    print(m.id)
```

Update `hf_repo` in `registry.yaml` if the correct repo differs from the default.

## Manual Registry Editing

`registry.yaml` is generated by `select_models.py` but can also be edited by hand.
Required fields per model entry:

```yaml
- id: <short-id>          # used by benchmark runner scripts
  hf_repo: <org/repo>     # exact HuggingFace repo ID
  revision: <sha>         # 40-char commit SHA — REQUIRED
  family: <family>        # qwen | llama3 | phi4 | glm4 | deepseek
  tier: <tier>            # small | mid | large | xl | reference
  params_b: <float>       # parameter count in billions
  precision: <precision>  # fp16 | fp8
  vram_gb_estimate: <int>
  context_length: <int>
  vllm_extra_args: "<string>"
```

Optional fields: `quantized_path`, `architecture`, `notes`.
